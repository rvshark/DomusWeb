SourceID como alternativa

Em dados IMS, o campo <sourcedid> representa o código ID persistente para uma pessoa como usado no sistema original. O campo <userid> é um campo separado que deverá conter o código ID usado pelo usuário para fazer o login.

Em muitos casos estes dois códigos podem ser o mesmo - mas nem sempre.

Alguns sistemas de informação de estudantes falham ao enviar o campo <userid>. Se for este o caso, você deve ativar esta opção para permitir o uso do <sourcedid> como ID do usuário no Moodle.

Em caso contrário deixe esta opção desativada.
